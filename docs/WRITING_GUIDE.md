# GitGrove 라이팅 가이드 (검수용 요약)

> 출처: Claude Design 프로젝트 "GitGrove" — `GitGrove 라이팅 가이드.html`.
> 원본(시각 포함): [`docs/writing-guide.html`](./writing-guide.html).
> **이 문서는 PM·QA가 UI/메시지 텍스트를 검수할 때 쓰는 기준이다.** 사용자에게 보이는 모든 한국어 텍스트(버튼·라벨·토스트·에러·빈 상태·툴팁·모달 문구)는 아래 규칙을 따라야 한다.

## 브랜드 한 줄
- **커밋 하나, 새싹 하나.**
- GitGrove = macOS용 다크모드 Git GUI. 저장소가 정원의 나무처럼 자라고, 마스코트 **그루(Geuru)**가 표정으로 상태를 전한다.

## Voice — 4원칙 (가장 중요, 모든 카피의 판정 기준)

1. **한국어 먼저, 짧게**
   - 한글이 기본. 버튼·라벨은 **명사형으로 짧게**. 주어(당신·제가)는 생략.
   - 개발자용 표기(phase, WebSocket, origin, main 등)만 영어로 남긴다.
   - ⭕ `변경을 골라 심어요`  ❌ `당신의 변경사항을 선택하세요`

2. **따뜻하지만 호들갑 없이**
   - 친한 방장이 말하듯 담백하게. **과장 형용사·느낌표 남발 금지.**
   - 레트로/아케이드 감성은 그루 표정·픽셀이 맡고, 글은 차분하게.
   - ⭕ `머지 완료 · main 최신화`  ❌ `머지 성공! 지금 바로 확인하세요!!`

3. **상태는 그루가 표정으로**
   - 글로 다 설명하지 않는다. 진행·완료·충돌은 그루 표정(think · merge · conflict · sleepy)이 먼저 전하고, 글은 **사실만 짧게** 받친다.
   - ⭕ `충돌 발생 · 3 files`  ❌ `앗! 충돌이 발생했어요. 당황하지 마세요`

4. **가운뎃점(`·`)으로 잇기**
   - 상태 조각은 쉼표·파이프 대신 가운뎃점 `·`로 잇는다.
   - 숫자는 서양 숫자, 99 초과는 `99+`. 에러는 **사유 먼저, 코드는 뒤에**.
   - ⭕ `origin · main · 방금 동기화됨`  ❌ `origin, main, 방금 동기화됨`

## Microcopy 표준 (그대로 차용)

- **버튼**: 짧은 명사. 영어는 Sentence case. 예: `Clone` · `열기` · `커밋` · `머지` · `승인` · `스태시`
- **빈 상태**: 현재 상태 + 다음 행동 제안.
  - `아직 심은 나무가 없어요` / `저장소를 Clone 하거나 폴더를 열어 보세요`
  - `아직 커밋이 없어요` / `첫 커밋을 심으면 그루가 자라요`
- **에러**: 담백한 명사 문장, 사유 먼저 + 코드 뒤.
  - `인증이 필요해요 · 401`
  - `저장소를 찾을 수 없어요 · 404`
  - `네트워크 오류 · 503`
- **토스트**: 그루 표정 + 사실 한 줄.
  - `머지 완료` / `main 최신화 · 12 files`
  - `충돌 발생` / `3 files · 해결이 필요해요`

## Feature copy 톤
화면 소개 한 줄은 "무엇을 하는지"보다 **"무엇이 좋아지는지"**를 말한다.
- 히스토리 · 스테이지&커밋 · Diff 탐색기 · PR/MR 리뷰 · 저장소/그로브 · 동기화/Clone

## Do / Don't

**Do**
- 해요체로 차분하게 · 명사형 버튼
- 상태 조각은 가운뎃점 (`name · room · phase`)
- 감정은 그루 표정에 맡기고 글은 사실만
- 에러는 사유 먼저 (`네트워크 오류 · 503`)
- 정원 · 나무 · 새싹 은유는 **자연스러울 때만**

**Don't**
- 느낌표 남발, `"최고의"` `"지금 바로"` 같은 과장
- 장식용 이모지 (기능 글리프만 허용)
- 한글 대문자 강조 · `당신` · `제가`
- 뻔한 버튼 설명 툴팁
- 은유 과용 — 나무 비유를 모든 문장에 욱여넣기

## 검수 체크리스트 (PM·QA용)

사용자에게 보이는 텍스트가 추가/변경되면 다음을 본다:
- [ ] 느낌표·과장 형용사("최고의", "지금 바로", "성공!")가 없는가
- [ ] 버튼/라벨이 명사형으로 짧은가, 주어(당신/제가)를 빼었는가
- [ ] 상태 조각을 쉼표/파이프 대신 가운뎃점 `·`으로 이었는가
- [ ] 에러가 "사유 먼저 · 코드 뒤" 형식인가
- [ ] 빈 상태에 다음 행동 제안이 있는가
- [ ] 장식용 이모지가 없는가 (기능 글리프만)
- [ ] 숫자 표기(서양 숫자, `99+`)가 맞는가
- [ ] 토스트/상태 문구를 그루 표정이 받치고 글은 사실만인가
